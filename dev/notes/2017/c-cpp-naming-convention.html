Another talking about C/C++ naming convention

Category: Programming Language. Tag: C/C++

Announcement:

This is maybe a topic that quite a few people are sick of reading, just like me. Yes, I have read too many discussions, or debates actually, since almost a decade ago, when I started to write the first line of C++ code. It seems there is no large devergence in the naming convention of C programming. Lower case is used naturally. Some people use more under-scores, and some use more abbreviations, but few assert any one is better. While in C++ the situation get complicated.

Anyway, I thought this is not a topic that deserves much more discussions. It does not mean I think this is not a serious issue. Actually, I think it is very important for a programmer to understand the naming conventions of predominant languages and follow one if necessary. The naming convention is more about preference for a programmer, while a rule for a project. Nothing is perfect, so is any naming convention. I have followed a consistent naming convention in the past years in my own projects, but I decided to change something recently. Again, I do not think this is a choice of better or not. It is just a change of preference. I do not intend to introduce or discuss the details about C++ naming conventions here. I just record my change and my thinking about it.

I don't like all lowercase name.
